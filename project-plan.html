개요 범위 기능 데이터 기술 산출물

Project Plan

악궁확장 치료 과정을 매일 정확하게 기록하는 환자 전용 앱

환자는 고유 코드로 로그인해 장치 모델, 회전수, 통증, 메모를 기록하고 병원은 누적 확장량과 이상 증상 보고를 확인하는 통합 관리 시스템을 구축합니다.

핵심 목표

치료 기록의 누락을 줄이고, 모델별 확장량 계산을 자동화하며, 이상 증상 발생 시 병원이 빠르게 확인할 수 있는 구조를 만든다.

Service Scope

서비스 구성은 환자 앱, 병원 관리자, 백엔드 서버 세 축으로 설계합니다.

01

환자용 모바일 앱

iOS와 Android에서 사용할 수 있는 앱. 고유 코드 로그인, 장치 모델 선택, 일일 기록 입력, 이상 증상 보고를 제공한다.

02

병원 관리자 웹

환자 목록, 환자별 장치 모델, 누적 확장량, 이상 증상 사진과 메시지를 확인하는 최소 관리자 페이지를 제공한다.

03

백엔드 및 인프라

환자 코드 인증, 치료 기록 저장, 사진 업로드, 관리자 데이터 제공, 병원 알림 연동을 담당한다.

1

코드 로그인

병원에서 발급한 환자 고유 코드만 입력한다.

2

모델 선택

MSE, KBE 등 치료 장치 모델을 최초 1회 선택한다.

3

일일 기록

회전수, 통증, 메모를 입력하고 확장량을 자동 계산한다.

4

증상 보고

사진과 메시지로 병원에 이상 증상을 전송한다.

5

병원 확인

관리자가 환자별 진행 상황과 보고 내용을 확인한다.

Core Features

핵심 기능은 치료 기록을 정확하게 남기고 병원이 즉시 확인하는 데 집중합니다.

환자 앱 기능

  • 아이디와 비밀번호 없이 병원 발급 고유 코드로 로그인
  • MSE1, MSE2, NEW MSE2, MSE3, KBE, KBE2 모델 중 단일 선택
  • 모델별 1회 확장량과 총 확장 가능량 자동 표시
  • 날짜, 확장 횟수, 회전수, 통증 단계, 메모 입력
  • 당일 확장량과 누적 확장량 자동 계산
  • 사진 첨부와 텍스트 메시지로 이상 증상 보고

병원 관리자 기능

  • 등록된 전체 환자 목록과 환자별 고유 코드 확인
  • 환자별 악궁확장기 모델 및 누적 확장량 확인
  • 환자가 전송한 이상 증상 사진 및 메시지 확인
  • 신규 보고 여부와 확인 상태 관리
  • 필요 시 병원 서버 또는 이메일 알림 연동
  • 추후 환자별 치료 리포트와 그래프 확장 가능

Data Design

데이터는 환자, 장치 모델, 확장 기록, 이상 보고를 기준으로 단순하고 명확하게 나눕니다.

데이터 주요 항목
환자 정보 환자 ID, 고유 코드, 장치 모델, 최초 로그인 여부
장치 모델 모델명, 1회 확장량, 총 확장 가능량, 안내 문구
확장 기록 날짜, 확장 횟수, 회전수, 통증, 메모, 누적 확장량
이상 보고 보고 일시, 증상 메시지, 첨부 사진, 확인 상태

운영 기준

의료 기록 성격의 데이터이므로 환자 식별 정보는 최소화하고, 사진 파일은 별도 스토리지에 저장하며, 서버에는 접근 가능한 파일 경로와 상태값만 관리한다.

고유 코드 인증 기록 자동 계산 사진 스토리지 관리자 확인 상태

Recommended Stack

턴키 개발 기준으로 앱, 서버, DB, 파일 저장, 배포까지 한 업체가 책임지는 구조를 권장합니다.

A

모바일 앱

React Native 또는 Flutter. iOS와 Android를 하나의 코드베이스로 개발해 일정과 비용을 줄인다.

B

백엔드 서버

Node.js / NestJS 기반 API 서버. 인증, 기록 저장, 파일 업로드, 관리자 데이터 제공을 담당한다.

C

DB 및 스토리지

PostgreSQL과 S3 호환 스토리지. 구조화 데이터와 증상 사진 파일을 분리해 관리한다.

Phase 1

상세 기획

화면 구조, 기능 정의, 데이터 구조, 관리자 범위를 확정한다.

Phase 2

디자인

환자 앱 UI와 병원용 최소 관리자 화면을 설계한다.

Phase 3

개발

앱, API 서버, DB, 파일 업로드, 관리자 웹을 구현한다.

Phase 4

배포

클라우드 인프라에 배포하고 운영 테스트를 진행한다.

최종 목표

환자는 자신의 악궁확장 치료 과정을 매일 정확히 기록하고, 병원은 환자별 누적 진행 상황과 이상 증상을 빠르게 확인할 수 있는 실사용 가능한 치료 기록 관리 시스템을 구축한다.

본 문서는 업체 검토용 요약 설계서입니다. 실제 착수 전에는 모델별 확장량 기준, 병원 업무 프로세스, 개인정보 처리 범위를 최종 확인해야 합니다.
